        if (androidTts != null) return

----- Sous-problème B : faux positifs wave -----

Cause racine : l'algo nourrit l'historique de wrist.x() DÈS
qu'une main est en vue. Avec les seuils relâchés du #8 et un
bon taux de détection (~25%, vu dans les logs), n'importe
quelle main qui dérive un peu déclenche un wave.

Fix structurel : ajouter un filtre "open-palm" avant de
nourrir l'historique. On n'enregistre la position du poignet
QUE si les 4 doigts (index, majeur, annulaire, auriculaire)
sont étendus. Le pouce est volontairement ignoré (moins
fiable, et un wave avec pouce replié reste un wave).

Critère "doigt étendu" : la distance 2D (wrist → fingertip)
est plus grande que la distance (wrist → PIP joint). Marche
quelle que soit l'orientation du poignet, plus robuste qu'un
test sur l'axe Y seul.

Indices MediaPipe utilisés :
  - WRIST = 0
  - Index (PIP=6, TIP=8), Middle (10, 12), Ring (14, 16),
    Pinky (18, 20)

Avec ce filtre actif, on peut maintenant resserrer les seuils
de mouvement sans manquer de vrais waves.

    private fun handleResult(result: HandLandmarkerResult) {
        if (result.landmarks().isEmpty()) return
        handsDetected++
        if (!firstHandLogged) { ... }
        val landmarks = result.landmarks()[0]
        // Only feed the history when fingers are extended (real "waving hand", not a fist
        // or a hand resting in frame). This kills the "presence detector" false-positives.
        if (!isOpenPalm(landmarks)) return

        val wristX = landmarks[WRIST].x()
        val now = System.currentTimeMillis()
        synchronized(wristXHistory) {
            wristXHistory.addLast(now to wristX)
            ...
            if (detectsWaveNow() && now - lastWaveTimeMs > COOLDOWN_MS) {
                lastWaveTimeMs = now
                wristXHistory.clear()   // avoid immediate re-trigger on the next frame
                Log.i(TAG, "Wave detected!")
                onWaveDetected()
            }
        }
    }

    private fun isOpenPalm(landmarks: List<NormalizedLandmark>): Boolean {
        if (landmarks.size < 21) return false
        val wrist = landmarks[WRIST]
        fun dist(a: NormalizedLandmark, b: NormalizedLandmark): Float {
            return hypot(a.x() - b.x(), a.y() - b.y())
        }
        return FINGER_PIP_TIP.all { (pipIdx, tipIdx) ->
            dist(wrist, landmarks[tipIdx]) > dist(wrist, landmarks[pipIdx])
        }
    }

----- MainPage.kt -----

Ajout d'une méthode showHelloOverlay() appelée dans le
callback du wave (juste après RobotController.speak). Un
Runnable différé masque l'overlay après 2 secondes. On
removeCallbacks d'abord pour éviter qu'un wave qui retombe
pendant l'affichage le coupe court.

--- AVANT (callback wave) ---

    waveRecognizer = WaveGestureRecognizer(
        context = requireContext(),
        lifecycleOwner = viewLifecycleOwner,
    ) {
        requireActivity().runOnUiThread {
            RobotController.speak("Hello!")
        }
    }

--- APRÈS ---

    waveRecognizer = WaveGestureRecognizer(
        context = requireContext(),
        lifecycleOwner = viewLifecycleOwner,
    ) {
        requireActivity().runOnUiThread {
            RobotController.speak("Hello!")
            showHelloOverlay()
        }
    }

    private val helloHideRunnable = Runnable {
        view?.findViewById<View>(R.id.helloOverlay)?.visibility = View.GONE
    }

    private fun showHelloOverlay() {
        val overlay = view?.findViewById<View>(R.id.helloOverlay) ?: return
        overlay.visibility = View.VISIBLE
        overlay.removeCallbacks(helloHideRunnable)
        overlay.postDelayed(helloHideRunnable, 2000L)
    }

Conséquence : même si le TTS échoue (cas d'origine sur la
build .test avant le fallback Android), l'utilisateur a une
confirmation visuelle que le wave a été reconnu.
